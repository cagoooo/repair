### Phase 0.11.0: 新學期教室配置安全換版
- **發布日期**: 2026/07/22
- **主要功能**:
  - [x] **圖片優先、PDF 選用**: 支援 PNG、JPG、GIF、WebP 與 PDF 選頁轉圖，單檔上限 10MB。
  - [x] **安全 OCR 後端**: Vision API 改由獨立 Cloud Function 呼叫，包含管理員驗證、每人每小時 5 次額度與安全輸入限制。
  - [x] **固定編號資料關聯**: 新舊學期以不變的教室編號銜接，避免班級名稱變更造成歷史報修失聯。
  - [x] **匯入差異預覽**: 套用前列出新增、變更、未變與 OCR 未辨識但保留的教室；重複編號或不合法資料會阻止套用。
  - [x] **自動備份與復原**: 每次正式儲存前自動備份目前配置，管理員可從歷史版本復原。
  - [x] **學年度／學期標籤**: 可標記如「115學年度第1學期」，並顯示目前配置版本。
  - [x] **跨系統部署保護**: 保留共用 Firebase 專案中的 3D 畫廊、行政系統及 `companion` 獨立資料庫規則。
  - [x] **正式資料基準稽核**: 101 間教室、48 筆報修、0 筆失聯；已將 `C640 幼兒園` 列為必須保留或人工確認項目。

### 115學年度換版標準流程
1. 填寫學年度／學期標籤，例如「115學年度第1學期」。
2. 上傳清晰的教室配置圖片；也可上傳 PDF 後選擇正確頁面。
3. 進入編輯畫面執行 AI 辨識。
4. 檢查新增、名稱變更、未辨識保留與重複編號。
5. 套用比對結果，視需要微調教室框位置。
6. 點擊「儲存設定」正式發布；系統會先備份舊配置。
7. 發布後抽查地圖、教室報修入口及歷史報修連結；必要時從歷史版本復原。

## 🧭 未來優化改良與可開發功能

> 優先級定義：P0 為 115 學年度正式換版前最值得先補齊的安全保障；P1 為換版可用後，能明顯提升效率、可維護性與管理價值的功能。

### P0：正式換版安全與可靠性

- [ ] **P0-1 新學期換版精靈**（中等）
  - 將「標記學期 → 上傳 → OCR → 差異確認 → 校正 → 發布 → 抽查」整合成有進度的步驟式流程。
  - 未完成必要步驟時禁止發布；離開頁面前提醒尚未儲存的變更。
  - 完成條件：管理員不需要記憶操作順序，也不會只換圖片卻忘記套用教室資料。

- [ ] **P0-2 發布前完整性檢查與硬性阻擋**（中等）
  - 檢查教室總數異常、重複／空白編號、座標超界、過小區塊、圖片比例劇烈變動、歷史報修失聯與未確認保留項目。
  - 嚴重錯誤禁止發布，警告則要求管理員逐項確認。
  - 完成條件：發布畫面能產生清楚的紅／黃／綠檢查報告。

- [ ] **P0-3 使用 115 學年度實圖進行正式演練**（小型，需取得實圖）
  - 用真正的 115 學年度配置圖完成一次不發布或測試版匯入，記錄 OCR 命中、誤判、漏判與人工修正項目。
  - 重點抽查直排編號、廁所編號、幼兒園 `C640`、密集教室區及所有有歷史報修的教室。
  - 完成條件：產出演練報告與可重複使用的 OCR 回歸測試樣本。

- [ ] **P0-4 動態管理員上傳權限一致化**（中等）
  - 目前 Firestore 可使用動態管理員名單，但 Storage 地圖上傳仍限主要管理員帳號。
  - 建議改由受保護的 Cloud Function 上傳，或使用 Firebase Custom Claims 統一管理員權限。
  - 完成條件：授權管理員可完整執行換圖、OCR、儲存與復原，未授權帳號則全部拒絕。

- [ ] **P0-5 低信心／未辨識教室人工確認佇列**（中大型）
  - 將 OCR 低信心、名稱疑似異常、找不到名稱及僅沿用舊座標的教室集中列出。
  - 提供「確認正確、手動修正、保留舊資料、暫不顯示」等明確處理方式。
  - 完成條件：所有疑慮項目都必須被人工處理後才能發布。

- [ ] **P0-6 共用 Firebase 規則自動回歸測試**（中等）
  - 為修繕、3D 畫廊、行政系統與 Storage 建立權限測試，並在部署前自動驗證。
  - 固定檢查 `companion` 獨立資料庫不在報修部署範圍內。
  - 完成條件：任何規則刪漏或權限放寬都會使 CI 失敗並阻止部署。

### P1：效率、管理與進階體驗

- [ ] **P1-1 新舊配置圖並排／透明疊圖比較**（中等）
  - 可調透明度查看教室框位移，並以顏色標示新增、移動、名稱變更與未辨識區域。

  - 以教室編號為主鍵批次更新班級名稱；支援匯出備查及錯誤列提示。

- [ ] **P1-3 教室資料批次編輯器**（中等）
  - 提供搜尋、篩選、多選與批次修改名稱、類別、顯示狀態，減少逐間點擊。

- [ ] **P1-4 歷史版本強化**（中等）
  - 增加縮圖、建立者、備註、差異統計、來源檔案與「復原前預覽」，使版本更容易辨認。

  - 報修單保留發生當時的學期與教室名稱快照，可比較不同學期的故障熱點、件數與平均修復時間。

  - 顯示每人／每日辨識次數、成功率、失敗原因、處理時間及額度阻擋紀錄，方便掌握成本與異常。

- [ ] **P1-7 配置檔案生命週期管理**（中等）
  - 清理取消操作、失敗上傳與過舊且未被版本引用的孤兒圖片／PDF，避免 Storage 長期累積。

- [ ] **P1-8 行動裝置編輯體驗**（中大型）
  - 加入觸控拖曳、雙指縮放、較大的調整控制點；或在手機明確切換成只檢查、不精修模式。

- [ ] **P1-9 配置發布通知與抽查清單**（小型）
  - 發布後通知管理員並顯示抽查項目：地圖載入、幾個代表教室、報修入口、列表回到地圖及版本備份。

- [ ] **P1-10 定期資料健康檢查與備份匯出**（中等）
  - 排程檢查重複編號、失聯報修與無效座標；提供 JSON／Excel 備份下載及異常通知。
